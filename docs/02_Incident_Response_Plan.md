# Incident Response Plan (IRP)
## Organization: [Company Name]
**Document Owner:** CISO / Security Operations Center (SOC) Lead
**Version:** 1.2

### 1. Overview
This document outlines the procedures the organization will follow to detect, respond to, and recover from cybersecurity incidents.

### 2. Incident Classification
- **Low:** Minor policy violations, isolated malware contained by EDR.
- **Medium:** Targeted phishing campaigns, degraded performance of internal systems.
- **High:** Confirmed data exfiltration, compromise of critical platforms, ransomware infections.
- **Critical:** Widespread systemic compromise affecting multiple clients or core infrastructure.

### 3. Phases of Incident Response
#### Phase 1: Preparation
- Continuous training of the Computer Security Incident Response Team (CSIRT).
- Regular table-top exercises simulating data breaches.

#### Phase 2: Identification
- Monitoring via SIEM tools.
- Triage of alerts generated by endpoint agents and network sensors.

#### Phase 3: Containment, Eradication, and Recovery
- **Containment:** Disconnecting affected subnets; blocking malicious IP addresses.
- **Eradication:** Removing malicious artifacts; patching the exploited vulnerability.
- **Recovery:** Restoring services from immutable backups; monitoring for reignition.

#### Phase 4: Post-Incident Activity
- Conducting a root-cause analysis (RCA).
- Updating policies and generating an after-action report within 72 hours of incident closure.
